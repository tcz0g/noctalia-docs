# Lockscreen widgets

Lockscreen widgets use the same widget types and per-type settings as [desktop widgets](/v5/desktop/widgets/). Layout is stored in a dedicated `[lockscreen_widgets]` section (not shared with `[desktop_widgets]` or `[lockscreen]` backdrop keys).

- Declarative config: `[lockscreen_widgets]` in `~/.config/noctalia/*.toml`
- GUI/editor overrides: `settings.toml` under your Noctalia state directory (same as desktop widgets)

```toml
[lockscreen_widgets]
enabled = true

[lockscreen_widgets.widget.clock_main]
type     = "clock"
output   = "DP-1"
cx         = 1600.0
cy         = 120.0
box_width  = 360.0
box_height = 120.0
rotation   = 0.0

[lockscreen_widgets.widget.clock_main.settings]
format = "{:%H:%M}"
```

When enabled, widgets render on each lock surface, over the same blurred and tinted wallpaper as the rest of the lock screen. To show a clock on the lock screen, add a **clock** widget.

Text widgets (clock, weather, media player, label, sysmon) support a per-widget `font_family` setting just like desktop widgets - empty inherits the shell font. See [Desktop widgets - Common settings](/v5/desktop/widgets/#common-settings).

## Edit mode

Edit mode is only available while the session is **unlocked**. The editor shows a dimmed full-screen preview with a movable **login box** preview on each monitor (one per output).

### Related IPC

For lockscreen widget edit-mode commands, see
[Surfaces → Lockscreen widgets](/v5/ipc/surfaces/#lockscreen-widgets).

You can also use **Settings → Security → Lock screen → Toggle Editor**.

### Controls

Same interaction model as desktop widget edit mode (move, rotate, resize, grid snap, center guides, toolbar), except the **login box** widget can only be moved (not resized or rotated). You can toggle the visibility of login boxes for individual monitors using the eye icon in the toolbar, provided that at least one login box remains visible across all monitors. Select it and open the settings inspector to adjust panel and input styling; **Reset to Defaults** at the bottom restores all login box settings. See [Desktop widgets - Edit mode](/v5/desktop/widgets/#edit-mode).

## Config format

The schema mirrors `[desktop_widgets]`: `enabled`, `schema_version`, optional `[lockscreen_widgets.grid]`, `widget_order`, and `[lockscreen_widgets.widget.<id>]` tables with `type`, `output`, `cx`, `cy`, `box_width`, `box_height`, `rotation`, `enabled`, and `[lockscreen_widgets.widget.<id>.settings]`.

Widget IDs generated in the editor use the `lockscreen-widget-` prefix. Login box placement uses fixed IDs `lockscreen-login-box@<output>` with `type = "login_box"` (not shown in the add-widget list).

### Login box settings

The login box uses the same inspector layout as other lockscreen widgets:

- **Widget** - login button visibility, password field opacity, corner radius, and text alignment (`show_login_button`, `input_opacity`, `input_radius`, `center_password_text`)
- **Background** - outer bar color, opacity, and corner radius (`background_color`, `background_opacity`, `background_radius`)

| Setting | Default | Range | Description |
| --- | --- | --- | --- |
| `background_color` | `surface_variant` | color role | Outer login bar fill |
| `background_opacity` | `0.88` | `0.0`–`1.0` | Outer login bar opacity |
| `background_radius` | `12` | `0`–`32` | Outer login bar corner radius (logical px) |
| `show_login_button` | `true` | bool | Show the check button beside the password field |
| `input_opacity` | `1.0` | `0.0`–`1.0` | Password field background opacity |
| `input_radius` | `6` | `0`–`32` | Password field corner radius (logical px) |
| `center_password_text` | `false` | bool | Center password dots and placeholder inside the field |

```toml
[lockscreen_widgets.widget."lockscreen-login-box@DP-1".settings]
background_color = "surface_variant"
background_opacity = 0.75
background_radius = 16.0
input_opacity = 0.9
input_radius = 8.0
```
